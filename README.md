![Platform](https://img.shields.io/badge/platform-Android-brightgreen)

**«Успеватель»** — мобильное приложение для интеллигентов, реализующее систему управления задачами по методике Василия Кислого, описанной в книге «Успеватель для интеллигентов». Это не таск-менеджер в привычном смысле — это **система личной продуктивности**, адаптированная для людей с нелинейным мышлением.

> **Успеватель — не планировщик. Это способ перестать чувствовать вину за несделанное и начать делать то, что реально важно.**

## 🧠 Философия

Методика Василия Кислого основана на парадоксальной, но работающей идее:

**Интеллигент не может быть дисциплинированным в классическом смысле.** Попытки загонять себя в жёсткие рамки приводят к чувству вины, выгоранию и откату.

Вместо этого «Успеватель» предлагает:

- **Элементарные действия** — любую задачу надо свести к одному конкретному шагу, который можно сделать прямо сейчас
- **4 зоны внимания** — вместо сотен проектов и папок
- **Револьверные проекты** — долгосрочные цели, которые «вращаются», как барабан револьвера: ты делаешь один шаг за раз, без срока и без чувства вины
- **Утренний регламент** — 5 шагов утром, которые настраивают мозг на продуктивный день
- **Недельный обзор** — дневник успеха вместо разбора ошибок

## 🏗 Архитектура приложения

Проект следует **Clean Architecture** с разделением на три слоя:

```
Kislo-GTD/
├── domain/          # Бизнес-логика, модели, use cases
│   ├── model/       # Task, CategoryType, TaskStatus
│   └── usecase/     # CaptureTaskUseCase, CompleteTaskUseCase, ...
├── data/            # Работа с данными
│   ├── db/          # Room (Entity, DAO, Database, TypeConverters)
│   │   ├── dao/     # TaskDao, WeeklyStatsDao
│   │   ├── entity/  # TaskEntity, WeeklyStatsEntity
│   │   └── converter/
│   └── repository/  # TaskRepositoryImpl, WeeklyStatsRepositoryImpl
└── presentation/    # UI (Jetpack Compose + MVVM)
    ├── theme/       # Material 3 тема, цвета, типографика
    ├── navigation/  # NavHost, маршруты
    ├── common/      # Переиспользуемые компоненты
    ├── inbox/       # ***IN — Входящие
    ├── day/         # **DAY — Сегодня
    ├── control/     # *CONTROL — Контроль
    ├── later/       # **LATER — Потом
    ├── maybe/       # >>MAYBE — Мечты
    ├── project/     # Проекты
    ├── taskdetail/  # Детальный просмотр задачи
    ├── review/      # Утренний и недельный регламенты
    ├── dashboard/   # Главный экран
    ├── capture/     # Быстрый захват (Share intent)
    └── widget/      # Home screen виджет
```

**Паттерны:** MVVM + Repository + Use Cases + StateFlow
**DI:** Hilt (Dagger)
**Асинхронность:** Kotlin Coroutines + Flow

## 📂 Категории задач

| Категория | Код | Описание |
|-----------|-----|---------|
| ***IN | Входящие | Сырые мысли, идеи, задачи из внешнего мира. Каждое утро разбираются в элементарные действия |
| **DAY | На день | Элементарные действия на сегодня. Группируются по субъектам (имя человека). Стволовые задачи — внизу, срочные — сверху |
| \*CONTROL | Контроль | То, что ты делегировал или ждёшь от других. Регулярно просматривается |
| **LATER | Потом | Дела, которые не нужно делать сегодня, но и забывать нельзя. Отсюда задачи возвращаются в DAY |
| >>MAYBE | Мечты | Желания, идеи, проекты, к которым ты, возможно, когда-нибудь вернёшься |
| Проекты | Капсом | Долгосрочные цели (например, ПЕРЕЕЗД, ДИССЕРТАЦИЯ). Каждая задача в проекте — элементарное действие |

## 🌅 Регламенты

### Утренний (5 шагов, ~15 минут)

| Шаг | Категория | Действие |
|-----|-----------|----------|
| 1 | ***IN | Разобрать входящие: каждую мысль превратить в элементарное действие |
| 2 | **DAY | Организовать день: перераспределить, что-то отложить, что-то взять в контроль |
| 3 | **LATER | Если есть место в DAY — подтянуть задачи из отложенных |
| 4 | *CONTROL | Проверить контроль: отметить выполненное, снять просрочки |
| 5 | Проекты | Быстрый обзор текущих проектов |

### Недельный (~30 минут)

- Просмотр статистики: сколько дел выполнено, сколько из них проектные
- Контрольный показатель: 25-30% проектных дел от общего числа
- Обзор раздела **>>MAYBE**
- Дневниковая запись: «За что я могу себя похвалить?»

## 🎯 Проекты и револьверные проекты

**Простой проект** — это последовательность элементарных шагов к конкретной цели.

**Револьверный проект** — это долгосрочная цель без срока, которая «вращается»: каждый день/неделю ты делаешь ровно один маленький шаг, не испытывая вины, что не сделал больше.

*Пример: Проект «ЗДОРОВЬЕ» — сегодня одно отжимание, завтра одно приседание. Не «Начать бегать по утрам 5 км».*

## 🔨 Сборка

```bash
# Клонировать
git clone https://github.com/kozvits/Kislo-GTD.git
cd Kislo-GTD

# Сборка debug APK
./gradlew assembleDebug

# Сборка release APK
./gradlew assembleRelease

# Запуск тестов
./gradlew testDebugUnitTest

# Проверка lint
./gradlew lint
```

**Android Studio:** Откройте корневую папку `Kislo-GTD` — IDE распознаёт Gradle-проект автоматически.

## ⚙️ CI/CD

GitHub Actions настроен на:

| Событие | Действие |
|---------|----------|
| Push в main/master | Сборка debug APK + lint + тесты |
| Push c ключами подписи | + Сборка signed release APK |
| Pull Request | Проверка сборки + lint |

**Для подписанного релиза** добавьте в Secrets репозитория:
- `KEYSTORE_BASE64` — закрытый ключ в base64
- `KEYSTORE_PASSWORD` — пароль хранилища
- `KEY_ALIAS` — алиас ключа
- `KEY_PASSWORD` — пароль ключа
